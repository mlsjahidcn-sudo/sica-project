## 问题概述

合作伙伴团队成员在上传文档时遇到 400 错误。

## 核心问题

### 1. 文档上传缺少 `student_id` 参数（主要问题）

**错误原因**：

- 前端 `handleFileUpload` 函数只发送 `application_id`、`document_type` 和 `file`
- API (`/api/documents`) 要求 `student_id` 作为必需字段
- 页面已经获取了 `application` 数据（包含 `students.id`），但没有使用

### 2. 改进用户体验

- 添加成功/失败通知提示
- 增强错误信息显示
- 显示上传进度

## 技术方案

### 修复策略

1. **前端修复**：在 `handleFileUpload` 函数中，从已获取的 `application` 对象中提取 `student_id` 并发送给 API

2. **数据流**：

```
application.students.id (students table id)
↓
formData.append('student_id', application.students.id)
↓
POST /api/documents
```

3. **增强错误处理**：改进前端错误提示，显示 API 返回的详细错误信息

### 修改文件

**`/src/app/(partner-v2)/partner-v2/applications/[id]/documents/page.tsx`**

修改 `handleFileUpload` 函数：

```typescript
// 当前代码（错误）:
formData.append('application_id', resolvedParams.id);

// 修复后:
formData.append('student_id', application.students.id);
formData.append('application_id', resolvedParams.id);
```

同时需要检查 `application` 对象是否已加载，避免在数据未准备好时提交。
